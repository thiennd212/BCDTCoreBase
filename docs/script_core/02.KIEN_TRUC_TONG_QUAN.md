### 2.1. MVP Technology Stack

| Component | Technology | Localhost Option |
|-----------|------------|------------------|
| **Frontend** | React 18 + TypeScript | npm run dev (Vite) |
| **UI Components** | DevExtreme | DevExpress license |
| **Backend** | .NET 8 Web API | dotnet run / IIS Express |
| **Database** | SQL Server | LocalDB / Express / Developer |
| **Cache** | Redis / MemoryCache | In-Memory (đơn giản hóa) |
| **Background Jobs** | Hangfire | SQL Server storage |
| **Real-time** | SignalR | WebSocket localhost |

### 2.2. Localhost URLs

| Service | URL | Mô tả |
|---------|-----|-------|
| React App (Dev) | http://localhost:3000 | Vite dev server |
| API (HTTP) | http://localhost:5000 | .NET Kestrel |
| API (HTTPS) | https://localhost:5001 | .NET Kestrel SSL |
| Swagger | https://localhost:5001/swagger | API documentation |
| Hangfire | https://localhost:5001/hangfire | Job dashboard |
| SignalR Hub | https://localhost:5001/hubs/notification | Real-time |

### 2.3. MVP Simplifications

| Aspect | Production | MVP (Localhost) |
|--------|------------|-----------------|
| Load Balancer | Azure Front Door | None (single instance) |
| Cache | Redis Cluster | MemoryCache hoặc Redis single |
| Database | SQL Server AG | LocalDB / Express |
| HTTPS | Azure managed cert | Self-signed / dev cert |
| Monitoring | App Insights | Console logs + Serilog file |
| Email | SMTP Service | MailHog / Papercut (dev) |
| File Storage | Azure Blob | Local file system |

## 7. Development Setup

### 7.1. Prerequisites

| Tool | Version | Download |
|------|---------|----------|
| Visual Studio 2022 | 17.8+ | visualstudio.com |
| .NET SDK | 8.0+ | dotnet.microsoft.com |
| Node.js | 20 LTS | nodejs.org |
| SQL Server | 2022 Express/Dev | microsoft.com |
| Git | Latest | git-scm.com |

### 7.2. Quick Start

```bash
# 1. Clone repository
git clone <repo-url>
cd BCDTCoreBase

# 2. Database setup
# Mở SQL Server Management Studio, chạy scripts theo thứ tự:
# sql/v2/01.organization.sql → 14.seed_data.sql

# 3. Backend
cd src/BCDT.Api
dotnet restore
dotnet run

# 4. Frontend (terminal mới)
cd src/bcdt-web
npm install
npm run dev

# 5. Truy cập
# Frontend: http://localhost:3000
# API: https://localhost:5001/swagger
# Login: admin / Admin@123
```

### 7.3. appsettings.Development.json

```json
{
  "ConnectionStrings": {
    "DefaultConnection": "Server=(localdb)\\MSSQLLocalDB;Database=BCDT;Trusted_Connection=True;"
  },
  "Jwt": {
    "SecretKey": "your-256-bit-secret-key-here-for-dev-only",
    "Issuer": "BCDT",
    "Audience": "BCDT-Users",
    "ExpiryMinutes": 60
  },
  "Hangfire": {
    "DashboardPath": "/hangfire"
  },
  "Logging": {
    "LogLevel": {
      "Default": "Debug"
    }
  }
}
```

│                                 │                                       │

│                                 │  5. Monitoring setup                 │

│  PRODUCTION (Cloud)                                                    │
│  ┌─────────────────────────────────────────────────────────────────┐   │
│  │ • Multi-region deployment                                       │   │
│  │ • SQL Server Always On AG                                       │   │
│  │ • Redis Cluster                                                 │   │
│  │ • Application Insights                                          │   │
│  │ • Azure managed certificates                                    │   │

### Những gì KHÔNG thay đổi khi migrate:

- ✅ Source code (100% tương thích)
- ✅ Database schema (44 bảng)
- ✅ Business logic
- ✅ API contracts
- ✅ Frontend code

### Những gì CẦN thay đổi:

- Connection strings → Azure SQL
- Cache config → Redis connection
- Logging → App Insights
- Secrets → Azure Key Vault
- CORS → Production domains
